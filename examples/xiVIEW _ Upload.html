How to submit a crosslinking dataset as mzIdentML

Uploading peptide identifications as mzIdentML is the recommended way to use xiVIEW. The submitted mzIdentML file must meet the following criteria:

  1. It is a valid mzIdentML 1.2.0 file. That is, it validates against the mzIdentML 1.2.0 schema.

  2. The mzIdentML files should be semantically valid. That is, the use of CV terms should conform to the requirements of the mzIdentML 1.2.0 specification document. An official semantic validator for mzIdentML 1.2.0 has been developed: https://github.com/HUPO-PSI/mzIdentML/releases. Some semantic validation errors do not prevent xiVIEW from processing the data.

  3. Every <DBSequence> element that describes a target protein MUST include the protein sequence in a <Seq> subelement.

If the accession attribute of target <dbsequence> elements is a valid uniprot accession then UniProtKB protein annotations will be available.


Limitations of xiVIEW when processing mzIdentML 1.2.0 files:

  • Information in mzIdentML files on scores and the passing of thresholds at higher levels of consolidation (unique peptide; crosslink level; PPI) are not processed, only the PSM level scores/threshold passes.

  • xiVIEW does not curently read FASTA files that were linked to by mzIdentML files, hence item (iii) above. If you upload a FASTA file alongside an mzIdentML file then the FASTA file gets ignored.

  • Doesn’t support the “combined spectra file type” from mzIdentML 1.2.0

How to submit a crosslinking dataset as CSV

Information on the CSV formats that xiVIEW supports can be found here.

If you submit a CSV file, then you will also have to manually enter the information on modification masses.

You may also submit a FASTA file containing the protein sequences (file extension must be '.fasta'):

  • If you do not provide a FASTA file, then your protein IDs must be valid UniProtKB accession numbers.
  • If you do provide a FASTA file, then your protein IDs must all match identifiers in the FASTA file.

If the protein identifiers in the CSV file are valid UniProtKB accession numbers then UniProtKB protein annotations will be available.

Peak lists (optional)

  • Only the peptide identifications file (mzIdentML or CSV) is required, but without uploading peak lists you won't be able to inspect the supporting spectra using xiSPEC.
  • Supported file formats are: mzML, mgf, and ms2 (& zip/gz archives of these). File extension must be '.mzML', '.mgf', '.ms2' or '.zip'.
  • To upload multiple peak list files zip them together.
  • These are the peak list files exactly as searched.
  • If peak list data is uploaded then it must be complete, i.e. all spectra identified must be present, or the upload process will result in an error.

  • mzML tips:
    • Filter out MS1 spectra to reduce file size and upload/parsing time. (e.g. 'MS level 2-' in MSconvert)
    • use centroided MS2 data! (e.g. use 'Peak picking' for profile data in MSconvert)

xiFDR

For results calculated in xiFDR, export them from xiFDR as mzIdentML and upload this to xiVIEW (not the xiFDR CSV export).

There is a 10GB size limit on uploaded files.
